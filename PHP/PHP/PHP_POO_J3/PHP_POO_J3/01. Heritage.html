1. Heritage

L'HERITAGE

(Comment hériter du patrimoine de sa classe mére)

L'héritage, c'est le mécanisme propre à la théorie Objet, par lequel on peut créer une nouvelle classe sans être obligé de réécrire tout son contenu (des propriétés et des méthodes).

On "clone" la classe d'origine en une nouvelle classe qui comporte TOUT LE PATRIMOINE de la classe d'origine.

En effet, le contenu de la classe que l'on clone se retrouve dans la nouvelle classe, sans besoin de le réécrire.

Mais alors, si les 2 classes sont des clones exacts, à quoi cela sert-il ?

Justement, c'est ici que l'Objet nous a apporté des changements fondamentaux :

Nous pouvons améliorer la nouvelle classe pour qu'elle nous fournisse les services précis donc nous avons besoin. ( Services qui étaient, évidemment, absents de la classe d'origine)

Comment améliorer une classe héritée ?

La nouvelle classe, celle qui hérite d'une autre classe, peut se voir dotée de nouvelles propriétés et méthodes.

On distingue alors 2 méthodologies pour "améliorer" le patrimoine de la nouvelle classe, selon qu'on lui ajoute un nouveau contenu homonyme d'un contenu existant, ou non.

  • Si une propriété ou une méthode existait déjà avec la même signature (nom + paramétres identiques), alors, on parle de surcharge.

  • Si la propriété et/ou la méthode que l'on rajoute n'existaient pas encore, on parle d'enrichissement.

Illustrons ceci par un exemple plus parlant.

Nous allons créer une classe Personne possédant un patrimoine propre. Puis nous allons créer une classe Stagiaire héritant de la classe Personne, et lui rajouter du nouveau patrimoine par surcharge et par enrichissement.

class Personne
{
	public string $Nom;
	public string $Prenom;
	
	public function __construct(string $nom, string $prenom) {
		$this->Nom = $nom;
		$this->Prenom = $prenom;
        echo "Je suis dans le constructeur de Personne<br>";
    }
	
	public function affiche(){
		echo "Cette Personne s'appelle $this->Nom $this->Prenom <br>";
	}
}

class Stagiaire extends Personne 		//(1)
{
	public string $Cours;		//(2)
	
	public function __construct(string $nom, string $prenom,string $cours) {	//(3)
		parent::__construct($nom, $prenom);
		$this->Cours = $cours;
        echo "Je suis dans le constructeur de Stagiaire<br>";
    }	
	
	public function affiche(){		//(4)
		echo "Ce stagiaire s'appelle $this->Nom $this->Prenom et il suit le cours $this->Cours <br>";
	}
	
	public function __toString(){			//(5)
		parent::affiche();			//(6)
	}
	
}


$Nico = new Stagiaire("Duflot","Nicolas","PHP");
$Nico->affiche();
$Nico->__toString();

Explications

Dans le code précédent, nous constatons ligne //(1) que la classe Stagiaire hérite (extends) de la classe Personne. Sans autre forme de procès (ou à l'insu de son plein grè), la classe Stagiaire se trouve affublée de tout le patrimoine de sa classe mère Personne

En ligne //(2) nous avons rajouté une patrimoine qui n'existait pas dans le patrimoine de Personne. Il s'agit donc d'un enrichissement.

En ligne //(3) nous avons rajouté une méthode constructeur, mais elle n'a pas la même signature que celle qui se trouve dans l'héritage (le constructeur de Personne possède 2 paramètres, celui de Stagiaire en a 3). C'est donc bien un enrichissement également.

Note : de façon quasi systématique, en PHP, les constructeurs de classes héritées ne sont que des enrichissements.

En ligne //(4) nous avons rajouté une méthode possédant exactement la même signature que celle provenant de l'héritage . Ici, il s'agit donc de surcharge.

En ligne //(5) nous avons rajouté une méthode qui n'existait pas encore dans le patrimoine de la classe obtenu par héritage. C'est encore un enrichissement.

En ligne //(6) nous appelons (dans une méthode d'un objet hérité, constatez-le) une méthode de la classe parente (c'est à dire de la classe dont elle hérite).

Notez bien précieusement la syntaxe de cet appel.

Précisions sur la surcharge

Notez dans l'exemple ci-dessus que la méthode affiche() de la classe Personne est public.

Il en est de même dans la classe Stagiaire.

Par surcharge ,il n'est pas possible de réduire la visibilité d'une propriété et/ou d'une méthode.

Il n'était donc pas possible (testez ceci) de décrire la méthode affiche() de Stagiaire ainsi :

...
	private function affiche(){		//(4)
		echo "Cette Personne s'appelle $this->Nom $this->Prenom et elle suit le cours $this->Cours <br>";
	}
...

Par contre, l'inverse est possible.

Nous aurions pu déclarer la méthode affiche() de Personne comme private, et celle de Stagiaire comme public.

La régle générale est que par surcharge, on ne peut pas réduire la visibilité du patrimoine par surcharge, on ne peut que l'augmenter ou le laisser en l'état.

Est il possible d'hériter de plusieurs classes ?

Hélas NON.

Contrairement à certains langages de haut niveau (C++ et ses dérivés), mais comme d'autres (JAVA, entre autres), il n'est pas permis d'hériter de 2 classes.

On dit donc que PHP utilise de l'héritage simple alors que C++ utilise de l'héritage multiple.

La raison primordiale en est que PHP ne propose pas de solution de résolution de l'incident consistant en la présence de méthodes homonymes en provenance de l'héritage des classes héritées en parallèle.

Dans ce graphisme, vous devez concevoir, que si l'héritage multiple était possible, les classes C et B contiendraient toutes les deux le patrimoine de la classe A.

Et donc la classe D, qui hériterait de B et de C, aurait, par héritage, 2 fois le même patrimoine (celui de A), ce que PHP ne sait pas gérer.

Encore du vocabulaire

  • Une classe qui n'hérite pas d'une autre classe est appelée meta classe, et quelque fois (moins usité) classe de base.
  • Une classe dont on hérite est appelée classe mére ou classe pére.
  • Une classe qui hérite d'une autre classe est appelée classe fille ou classe fils ou classe dérivée.
  • une classe fille spécialise sa classe mère.
  • une classe mère généralise ses classes filles.

Rappelez vous la partie modélisation :

On peut dire qu'une classe A hérite d'une classe B si on peut dire :

B est un A

Par exemple : un Stagiaire est une Personne donc : class Stagiaire extends Personne
